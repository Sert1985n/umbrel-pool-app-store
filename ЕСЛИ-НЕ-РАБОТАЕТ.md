# Если в Umbrel ничего не работает или магазин пустой

## 1. Что должно быть на GitHub

Репозиторий: **https://github.com/Sert1985n/umbrel-pool-app-store**

- В корне: `umbrel-app-store.yml`, `README.md`, папки приложений (`sert-umbrel-pool-pool-config`, `sert-umbrel-pool-poolui`, ноды и т.д.).
- В каждой папке приложения: `umbrel-app.yml` и `docker-compose.yml`.
- **Моя панель (poolui)** и **Pool Config (Web UI)** — в магазине; poolui без сборки (статика в репо).

Если на GitHub этого нет — правки делаются в папке `C:\Users\WIN-10\Desktop\Umbrel-Pool-Project` и отправляются командами: `git add -A` → `git commit -m "..."` → `git push`.

## 2. Как добавить магазин в Umbrel (точно)

1. В Umbrel откройте **Settings** (Настройки) → **Community App Stores** (или **App Store** → источники).
2. Нажмите **Add new store** / **Добавить магазин**.
3. Вставьте **ровно этот URL** (без пробелов в начале и в конце):
   ```
   https://github.com/Sert1985n/umbrel-pool-app-store
   ```
4. Сохраните. Должен появиться магазин **Umbrel Pool**.
5. Если магазин появился, но **список приложений пустой** или **нет приложения Pool Config**:
   - Нажмите **Update** / **Refresh** / **Обновить** у магазина «Umbrel Pool».
   - В разделе **SOLO CRYPTO MINING** — **«Моя панель — Pool Dashboard»** (иконка LTC) и **«Pool Config (Web UI)»** (иконка ETH).
   - Если не помогло — удалите источник и добавьте снова URL, затем снова **Update**.

## 3. Типичные причины «ничего не работает»

| Проблема | Что сделать |
|----------|-------------|
| Магазин не добавляется / ошибка | Проверить интернет с устройства Umbrel. URL без лишних символов: `https://github.com/Sert1985n/umbrel-pool-app-store` |
| Магазин есть, но приложений нет | Update/Refresh у магазина. Удалить источник и добавить снова. Перезагрузить Umbrel. |
| Приложение установилось, но не открывается | Открывать по порту: 8562 — Pool Config, 8560 — Моя панель (poolui), 8563 — MiningCore Web UI. |
| **Моя панель (poolui) не устанавливается** | Переустановите приложение в Umbrel. Статика из репо, сборка не нужна. |
| **Pool Config не видно в магазине** | Обновите магазин (Update/Refresh). В разделе SOLO CRYPTO MINING — «Моя панель — Pool Dashboard» (LTC), «Pool Config (Web UI)» (ETH). |
| **8560 не открывается (ERR_CONNECTION_REFUSED)** | Моя панель (poolui). Проверьте порт в Umbrel. Переустановите приложение. |
| **Pool Config (8562) не ставится** | Без привязки к MiningCore. Сборка из репо. Повторите установку; при необходимости: `mkdir -p /home/umbrel/.miningcore`. |
| **В панели 8562 нет моих монет** | См. **guides/COINS-JSON-SETUP.md**: загрузите coins.json, перезапустите MiningCore и Pool Config (8562), обновите страницу 8562. |
| **Setup DB / Generate Config** | Откройте **8563** (кнопка на 8562 или http://IP:8563). Там Refresh Master Coin List, Setup Database Schema, Generate Pool Config File. |
| На GitHub вроде пусто | Обновить страницу (Ctrl+F5). Убедиться, что открыта ветка **main**. Зайти в папки (например, `sert-umbrel-pool-miningcore-webui`). |

## 4. Проверка, что репозиторий на GitHub полный

Откройте в браузере:

- https://github.com/Sert1985n/umbrel-pool-app-store (главная)
- https://github.com/Sert1985n/umbrel-pool-app-store/blob/main/umbrel-app-store.yml (должен открыться файл)
- https://github.com/Sert1985n/umbrel-pool-app-store/tree/main/sert-umbrel-pool-pool-config (панель конфигурации)
- https://github.com/Sert1985n/umbrel-pool-app-store/tree/main/sert-umbrel-pool-poolui (ваша панель Pool Dashboard)

Если страницы открываются и видны файлы — репозиторий в порядке, проблема в Umbrel (п. 2–3). Если страницы 404 или пустые — тогда правки не попали в GitHub и нужно пушить из папки `Umbrel-Pool-Project` (см. п. 1).
