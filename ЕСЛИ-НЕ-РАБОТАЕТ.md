# Если в Umbrel ничего не работает или магазин пустой

## Как увидеть свою панель, не удаляя ссылку на магазин

Ссылка **https://github.com/Sert1985n/umbrel-pool-app-store** уже добавлена. Чтобы увидеть приложение **Pool Dashboard (моя панель)** без удаления и повторного добавления:

1. В Umbrel открой **App Store** (или **Settings** → **Community App Stores**).
2. Выбери магазин **Umbrel Pool** (или открой его).
3. Нажми **Update** / **Refresh** / **Обновить** / **Sync** у этого магазина. Umbrel заново скачает репозиторий с GitHub.
4. Зайди в раздел **SOLO Crypto Mining** — там должны быть **MiningCore**, **Miningcore Web UI**, **Pool Dashboard (моя панель)**. Твоя панель — последняя (своя иконка, порт 8561).
5. Если кнопки Update нет: обнови страницу Umbrel (F5), снова зайди в App Store и открой магазин Umbrel Pool — в части версий список подтягивается при открытии.
6. Если панели всё равно нет: перезагрузи Umbrel (перезапуск сервиса или устройства), затем снова зайди в App Store и нажми Update у магазина Umbrel Pool.

Удалять и заново добавлять ссылку не обязательно — достаточно обновить магазин (п. 3).

## 1. Что должно быть на GitHub

Репозиторий: **https://github.com/Sert1985n/umbrel-pool-app-store**

- В корне: `umbrel-app-store.yml`, `README.md`, папки приложений (`sert-umbrel-pool-poolui`, `sert-umbrel-pool-miningcore-webui`, ноды и т.д.). Панель Pool Config (8562) удалена.

- **Моя панель (poolui)** и **Pool Config (Web UI)** — в магазине; poolui без сборки (статика в репо).

Если на GitHub этого нет — правки делаются в папке `C:\Users\WIN-10\Desktop\Umbrel-Pool-Project` и отправляются командами: `git add -A` → `git commit -m "..."` → `git push`.

## 2. Как добавить магазин в Umbrel (точно)

1. В Umbrel откройте **Settings** (Настройки) → **Community App Stores** (или **App Store** → источники).
2. Нажмите **Add new store** / **Добавить магазин**.
3. Вставьте **ровно этот URL** (без пробелов в начале и в конце):
   ```
   https://github.com/Sert1985n/umbrel-pool-app-store
   ```
4. Сохраните. Должен появиться магазин **Umbrel Pool**.
5. Если магазин появился, но **список приложений пустой** или **нет приложения Pool Config**:
   - Нажмите **Update** / **Refresh** / **Обновить** у магазина «Umbrel Pool».
   - В разделе **SOLO CRYPTO MINING** — **«Pool Dashboard (моя панель)»** (порт 8561) и **«Miningcore Web UI»** (порт 8563). Это две разные панели с разными иконками.
   - Если не помогло — удалите источник и добавьте снова URL, затем снова **Update**.

## 3. Типичные причины «ничего не работает»

| Проблема | Что сделать |
|----------|-------------|
| Магазин не добавляется / ошибка | Проверить интернет с устройства Umbrel. URL без лишних символов: `https://github.com/Sert1985n/umbrel-pool-app-store` |
| Магазин есть, но приложений нет | Update/Refresh у магазина. Удалить источник и добавить снова. Перезагрузить Umbrel. |
| Приложение установилось, но не открывается | **8563** — родная MiningCore Web UI (Setup DB, Generate Config). **8561** — ваша WEB UI (Pool Dashboard). Две разные панели. |
| **Моя WEB UI (poolui) не устанавливается** | Переустановите приложение в Umbrel. Статика из репо (poolui из www), сборка не нужна. |
| **Панели не видно в магазине** | Обновите магазин (Update/Refresh). «Pool Dashboard (моя панель)» (8561) и «Miningcore Web UI» (8563). |
| **8561 не открывается** | Ваша панель (poolui). Проверьте порт в Umbrel. Переустановите приложение. |
| **«Something went wrong» при установке Pool Dashboard** | Umbrel ставит только готовые образы (без build). Образ публикуется в GHCR через GitHub Actions. После пуша в репо подожди 3–5 мин, пока соберётся образ: https://github.com/Sert1985n/umbrel-pool-app-store/actions → workflow «Build and push Pool Dashboard». Потом в Umbrel удали магазин, добавь снова, нажми Install. |
| **Setup DB / Generate Config** | Откройте **8563** (http://IP:8563). Там Refresh Master Coin List, Setup Database Schema, Generate Pool Config File. Панель 8562 удалена. |
| На GitHub вроде пусто | Обновить страницу (Ctrl+F5). Убедиться, что открыта ветка **main**. Зайти в папки (например, `sert-umbrel-pool-miningcore-webui`). |

## 4. Проверка, что репозиторий на GitHub полный

Откройте в браузере:

- https://github.com/Sert1985n/umbrel-pool-app-store (главная)
- https://github.com/Sert1985n/umbrel-pool-app-store/blob/main/umbrel-app-store.yml (должен открыться файл)
- https://github.com/Sert1985n/umbrel-pool-app-store/tree/main/sert-umbrel-pool-poolui (ваша WEB UI — Pool Dashboard)
- https://github.com/Sert1985n/umbrel-pool-app-store/tree/main/sert-umbrel-pool-miningcore-webui (родная MiningCore Web UI)

Если страницы открываются и видны файлы — репозиторий в порядке, проблема в Umbrel (п. 2–3). Если страницы 404 или пустые — тогда правки не попали в GitHub и нужно пушить из папки `Umbrel-Pool-Project` (см. п. 1).
